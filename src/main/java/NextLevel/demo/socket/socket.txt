모듈화가 끝난 socket service와 그를 사용할 다른 service들의 입장에서 생각했을 때
controller :: socket service.createNewSocket();
other service :: socket service.sendData(data);
이 2개의 함수가 잘 구현되어 있어야 한다고 생각한다 때문에 아애 socket service를 2개로 나누는 것은 다른 service에서 혼란을 초래할 수 있다
통합된 socket service 다른 service 입장에서 편하고 유지보수 하기 편할 것이다

하지만 socket service의 확장성이 너무 떨어진다
  -> 다른 socket service를 둔다 해도 socket service.sendData()의 인자 값인 SocketDataDto와 SocketType 값만 바뀌기 때문에 확장성에도 문제가 없다 판단
